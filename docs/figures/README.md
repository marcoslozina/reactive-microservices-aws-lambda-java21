## Figura 6: Thread-per-Request vs Event Loop

**Archivo:** `figure-6-thread-vs-eventloop.png`

**Especificaciones:** Ver `figure-6-specs.md` para detalles completos de diseño.

### Descripción

Esta figura compara visualmente dos modelos de procesamiento:

- **Thread-per-Request (Blocking):** Modelo tradicional donde cada request consume un thread dedicado
- **Event Loop (Non-blocking):** Modelo reactivo donde un único event loop maneja múltiples tareas
